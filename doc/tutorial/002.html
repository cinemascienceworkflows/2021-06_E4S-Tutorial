5. The execute script

The place to start investigating a workflow is the execute script, which has the steps that the workflow will execute:
  • setup (install)
  • run the workflow application
  • wait for completion of the run
  • postprocess results
  • wait for completion of postprocess
  • validate results
The execute script shows the steps that are run. In the case of a workflow run on a target HPC system, the script would look like this: 

#!/bin/bash

source bootstrap.env
source pantheon/env.sh

./sbang.sh setup/install-deps.sh
./sbang.sh run/run.sh
./sbang.sh run/wait_for_completion.sh
./sbang.sh postprocess/postprocess.sh
./sbang.sh postprocess/wait_for_completion.sh
./sbang.sh validate/validate.sh
When run on a supercomputer, the steps are executed by submitting an application job to a queue. The wait_for_completion steps then monitor the queue to provide feedback on the state of the job. This allows the workflow to run continuously until completion.

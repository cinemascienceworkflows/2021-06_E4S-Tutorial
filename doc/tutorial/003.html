6. Overview of Steps

The setup/install-deps.sh script:
  • create work area
  • install spack (if needed)
  • install dependencies
    • Optionally use the E4S cache to significantly decrease install times.
    • Note that this is a critical step when working on target HPC systems, as build times are significantly reduced when using a cached build.
  • optionally install an application
The run/run.sh script:
  • creates a run directory
  • copies files from the input directory
  • submits the script to a job queue


The postprocess/postprocess.sh script:
  • installs several viewers alongside the cinema databases
  • packages up the cinema databases and the viewers
The validate/validate.sh script:
  • checks the output against known results
